## 코치님의 조언
1. 5주라는 시간이 부족함. 인공지능, 백, 프론트가 병렬적으로 진행되어, 일이 한 파트에 몰리지 않도록 하기 데이터가 유동적으로 변할 수 있다는 것을 감안하고 어떤 방식으로 진행될지 논의 필요

2. **데이터 양 / 데이터 모델 확인 필요**  
한국어 데이터셋이 부족하고, 적합한 데이터셋을 찾기 어려움.   
한국어 pretrain된 모델이 있는지 확인하는 것도 필요  
만약 영어 데이터로 웹서비스를 진행해야 한다면 
타겟 유저가 한국 문화에 관심이 많은 외국인이라던지 또는 영어공부하고 싶은 사람들 등으로 설득력 있게!!<br/>(ex) 
영어 공부하고 싶은 사람들을 타겟으로  변경하여
한국어 모델이 없는 점을 극복  
세줄 요약 => api로 번역


3. 모델이 무거울 수록 좋은 결과가 나온다. 따라서 **pretrain 모델 사용 추천**   
- 지적 대비 => pretrain모델을 사용하되 활용 측면으로  fine tuning, 여러 pretrain model 조사 및 테스트, 앙상블 학습 등을 통해 다양한 시도를 해서 어필 필요

4. **얼마나 사용하고 싶은 서비스인지가 중요!!!!!!**
5. 어쩔 수 없이 보여지는 것이 평가요소 == 프론트 역할 중요

6. 프론트 다음으로 인공지능, 백의 퀄리티 중요 (퀄리티 향상을 위한 다양항 시도 예시: 데이터셋 크롤링, 적합한 임의 데이터 생성, 모델 구조 변경, 앙상블 학습)
7. **간단한 task라도 다양한 시도로 평가에 좋은 인상 남기기** -> 여러 개의 모델을 써봤는데 그 중에서 어떤게 제일 좋았다!!

## 아이디어에 대한 코치님의 조언
1. AI로 사용자가 작성한 블로그 글의 맥락을 파악하여, 적절한 단어 또는 문장을 추천해주는 서비스
- 고민할 요소: 적합한 데이터셋, 모델, 성능
- 학습 방법 예시: 블로그글 크롤링 + fine tuning, 레이블링된 블로그 이모티콘으로 감성분석

2. 북마크 공유 서비스
- 유용한 모델: 해시태그 모델(한국어 버전 존재 불확실), 번역 모델
* 추가 조언: 모델을 두고 아이디어를 생각해보는 것도 좋다

3. 향수 추천 서비스
- 필요한 데이터셋: 향수 구매 정보, 향수 후기 데이터
- 전통적인 방식으로 성분이나 선호도를 이용해서 구현 가능

4. 자소서 쓰기 서비스
- 필요한 모델: 중요 키워드 뽑기, 맞춤법 교정
- 필요한 데이터셋: 평가 내용으로 레이블링 된 공개 자소서 데이터셋
- 난이도가 있다

5. 기초한국어 교육 서비스
- 필요한 모델: 한국어 챗봇, 문법 교정 모델, 언어 번역 모델
- 학습 디렉팅은 어렵다

6. 법률정보 제공 서비스
- 법률 지식 도메인 필요
- 법률 데이터셋이 적합한 형식인지(우리가 원하는 데이터인지) 확인할 필요 있다
- 필요한 모델: 한국어 요약 모델

7. 비슷한 낱말 추천 서비스
- 필요한 모델: word2Vec(유사 단어 추천)
